# 项目总说明

本项目目标不是“全自动刷号”，而是把 B站内容变成可复用的知识和草稿：

```text
添加博主/关键词 -> 抓公开视频 -> 抓关键指标 -> 抓评论 -> 总结用户关注点 -> 生成日报/知识库资料 -> 人工审核
```

## 子项目定位

| 子项目 | 定位 | 当前用途 |
| --- | --- | --- |
| `bilibili-summary` | 视频总结生产工具 | 后续接模型 API，生成视频 Markdown 总结 |
| `bilibili-ai-bot` | B站 Agent 实验工具 | 只研究 Agent / 记忆 / 草稿，不跑真实互动 |
| `tools` | 本工作区安全包装脚本 | 只读采集公开视频、评论和指标，生成本地 CSV / Markdown |

## 代码管理方式

`bilibili-summary/` 和 `bilibili-ai-bot/` 是本地部署目录，已在总仓库 `.gitignore` 中忽略。这样做是为了避免把外部开源项目源码混进本仓库；后续更新它们时，按各自项目来源重新下载或单独管理。

## 当前边界

- 不使用主号。
- 不保存真实 Cookie。
- 不自动发评论。
- 不自动私信。
- 不自动点赞、投币、关注。
- 不自动写入 Obsidian 正文。

## 内容洞察目标

当前重点是把本项目做成“B站内容情报 / 选题研究系统”：

1. 维护一批长期观察的 AI 博主。
2. 按关键词搜索新视频和新博主。
3. 记录播放、弹幕、点赞、投币、收藏、分享、评论数等关键指标。
4. 抽样读取公开评论，分析用户在意价格、教程、部署、效果、替代工具还是资料链接。
5. 生成日报、选题池和可导入 Obsidian 的资料。
