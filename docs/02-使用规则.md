# 使用规则

## 默认模式

默认只读 / 草稿模式：

- 可以读取公开视频信息。
- 可以生成总结草稿。
- 可以生成选题草稿。
- 可以生成评论草稿。
- 不直接发到 B站。

`bilibili-ai-bot` 首次导入配置模块会自动生成默认 `config.json`，且上游默认主动行为偏激进。启动前必须确认本地 `bilibili-ai-bot/config.json` 来自 `config.dry-run.example.json`，并确认主动行为全部关闭。

## 禁止默认开启

- 自动点赞。
- 自动投币。
- 自动收藏。
- 自动关注。
- 自动评论。
- 自动私信。
- 自动拉黑。
- 自动发动态。

## 敏感信息

以下内容只允许放在本地忽略文件，不写入公开仓库、不发聊天：

- `SESSDATA`
- `bili_jct`
- `DedeUserID`
- `REFRESH_TOKEN`
- API Key
- 密码
- Cookie

## 什么时候暂停

后续 Agent 执行时，只有遇到这些情况才需要暂停问用户：

- 需要用户扫码登录。
- 需要验证码。
- 需要真实 Cookie / Token / 密码。
- 需要开启真实点赞、投币、评论、私信、关注。
- 需要把敏感信息写入 Obsidian。
- 需要删除或覆盖已有重要文件。
